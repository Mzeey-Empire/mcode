Mcode

Performant AI agent orchestration desktop app built with Electron + TypeScript. Drive long-running coding agents across multiple worktrees and threads, review diffs and commits, and switch between projects without losing context.

Quick start

bun install
bun run dev

Run bun run setup to bootstrap from a fresh clone. Run bun run doctor to verify all prerequisites are installed.

Installation

Mcode runs as a desktop app on macOS, Windows, and Linux.

Download the latest release from the GitHub releases page.

Installing from a nightly build

Per-build releases are published nightly to the nightly channel. To opt in, set mcode.update.channel = "nightly" in your settings.

mcode update --channel nightly

Supported agent harnesses

This repo is configured for four agent harnesses. All four read AGENTS.md, share the same Stop-hook verify, and load the Playwright MCP for visual verification.

  • Claude Code — .claude/settings.json
  • Cursor — .cursor/hooks.json
  • Codex — .codex/hooks.json
  • OpenCode — .opencode/opencode.json

All four harnesses expose the same six commands: /verify, /verify-e2e, /verify-e2e-desktop, /demo, /demo-desktop, /review-pr.

The narrative pipeline classifies model thoughts text and thinking blocks separately, with the Whisper redesign surfacing prose-first content and rail-only nesting for sub-agents.

State 5 · open & reveal · existing desktop IPC

Mcode doesn't host its own code editor, but the desktop bridge already exposes openInEditor(editorId, path) and openInExplorer(path) (see OpenInEditorMenu.tsx). The rail's Open button surfaces a small picker anchored to its left, listing only the editors useInstalledEditors() detected on the user's machine. Reveal is a single-action button that opens the OS file manager at the file's directory. No new IPC is needed — both are wired today, just not yet surfaced in the diff view.
