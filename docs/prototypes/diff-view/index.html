Four ways the file row can stop misfiring.

Today, expanding a file diff makes the whole row a single button — and the Markdown preview toggle is a span pretending to be a button, nested inside that outer button. Misclick the icon and the row collapses. These four prototypes propose distinct fixes, each preserving the app's editorial / typeset feel.

What's broken

  • Invalid DOM — interactive content nested inside <button> (the MD preview toggle is a span role="button" wrapped by the row's button).
  • Hit target ~15px — an 11px icon with 2px padding, surrounded by the row's collapse area. Miss it, lose your place.
  • Fragile event handling — every inline action needs e.stopPropagation() just to survive.
  • No tab order for inline controls — accessing the preview toggle by keyboard requires shift-tab through the chevron, the filename, the stats…
  • One row, three intents — disclosure, navigation, and per-file actions are all crammed into the same control.
V1

Split Header

The row keeps single-click disclosure. When expanded, a thin secondary toolbar appears between the row and the diff content with real buttons (Raw/Rendered, Copy path, Open externally, Collapse).

Familiar Low risk Discoverable
V2

Side Rail

The row becomes pure disclosure (chevron + filename + stats). All per-file actions live in a vertical rail that appears to the right of the expanded diff content, with 28px hit targets and labels on hover.

Spatial Pencil-clean rows Wide panels only
V3

Segmented View Picker

Preview lives inside the diff content as a proper role="radiogroup" segmented control (Diff · Rendered · Side-by-side). The header row never owns toggles. Arrow keys cycle modes.

Editor-native Keyboard cycles Per-file mode
V4

Command Bar

A floating action bar slides in on hover / focus, pins when the row expands, and surfaces single-letter shortcuts inline (P preview, C copy, O open, Esc collapse).
