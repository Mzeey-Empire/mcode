V3 · Segmented View Picker

The toggle moves inside the file content as a proper segmented radiogroup — Diff · Rendered · Side-by-side — paired with a tiny keyboard hint. Arrow keys cycle, the selection persists per file, and the header row never owns toggles. Disabled modes (e.g. Rendered on a non-Markdown file) grey out but stay visible to teach the affordance.

  • DisclosureRow click = expand only
  • Moderole="radiogroup" inside content
  • Hit target≥ 70 × 28 px per segment
  • Keyboard←/→ cycles, D/R/S jumps
  • DiscoveryAlways visible when expanded
  • Vertical cost~46 px per expanded file
The key idea: mode is a property of the file content, not the file header. Mirrors how editors handle this (CodeMirror, GitHub's diff/rich toggle, MermaidBlock's source/render). Click target is wide and labelled, never an icon.
